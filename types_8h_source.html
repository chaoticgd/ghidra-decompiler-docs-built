202 /* In order to have a little more flexibility in integer precision vs efficiency,
203  we subdivide the integer types into three classes:
204 
205  Small integers: Integers that never come close to overflowing their (machine word)
206  precision. We will always use int4 or uint4 (or smaller) for
207  these so that the precision is explicitly given.
208 
209  Machine word integers: These integers exactly match the largest precision that
210  will fit in a general purpose register. They should be
211  used exclusively by in implementations of larger
212  precision objects. Use intm or uintm
213 
214  Big integers: These are intended to be arbitrary precison integers. However
215  for efficiency, these will always be implemented as fixed precision.
216  So for coding purposes, these should be interpreted as fixed
217  precision integers that store as big a number as you would ever need.
218 */
219
